怎么读这个看板？

每个 app 一张联系表（6~8 个核心页面拼成一张，带序号·标题·路由标注）+ 该 app 的设计治理指标。切换上方主题/视图看不同变体。

  • 健康分 A–F（0–100）：单一质量信号 = 无障碍 35% · Token 卫生 30% · 共享组件采用 20% · 页面覆盖 15% 加权。A≥90 B≥80 C≥70 D≥60 F<60。
  • 捕获覆盖：配置的页面截到了几张（缺=没截到，需排查）。
  • 无障碍：页内自动查「图无 alt / 控件无可访问名 / 输入无标注」的通过率。
  • 共享组件采用：源码里 @life-os/* 共享 UI import 占「共享+本地组件 import」的比例（越高越少造轮子）。
  • 样式债务：硬编码色值/字号/动效等应改成 Design Token 的存量数（越低越好），附整改方向。
  • 代码变更影响：自上一次生成到现在，这个 app 改了哪些文件/页面。
  • 整改路由：同类问题跨多个 app = 系统性，该在 Token/共享 层修；只在少数 app = 局部，局部改即可。
  • ▲/▼：相对上一代生成的变化（回归=红）。数据全部由脚本从真实源码/截图计算，缺失即不显示。

机器可读：manifest.json · llms.txt（AI agent 可直接读，无需 OCR 截图）。
