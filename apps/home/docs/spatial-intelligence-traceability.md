# HomeOS 空间与整理智能 — 可追溯性矩阵

> 计划 §0 的前置交付物。目的：把《Spatial & Organizing Intelligence Spec》逐条映射到
> **现状 / 本次新增 / 延后 / 测试**，据此核实「现有系统已实现约 80% 方法论」到底是
> 接通了产品行为，还是只有底层组件存在但未接线。
>
> 「现状」列的判定标准：
> - **已接通** = 产品行为端到端可见（有引擎 + 有消费方 + 有测试）
> - **部分** = 底层组件在，但产品语义未接通或不完整
> - **无** = 不存在
>
> 关联计划：`~/.claude/plans/homeos-plan-federated-canyon.md`（v2）。规则 ID 前缀：
> FN 功能真源 / SF 表面 / EN 包络 / PET 宠物 / CP 容量 / OWN 归属 / PH 阶段 / SAFE 硬安全 / PHOTO 照片。

## 小结（80% 判定）

- **已接通（现有，不重建）**：整理阶梯与指导（PH-01/02）、按几何归属（OWN-01/02/04）、
  门开合与动线/净空/堵门/瓶颈几何、退火求解器三方案、杂乱评分、VLM 房间识别与多角度身份、
  事件学习。→ 方法论骨架属实约 80%。
- **部分（有组件未接语义）**：功能真源、表面类型、椅子/门包络细分、宠物（仅手填 relations）、
  取物摩擦。
- **无（净新增）**：意图 vs 观察分离、`prohibited-storage` 强制清空、定性容量/满载、
  多风险×容器、五阶段分区状态机、硬安全规则（电源/热/承重）、前后照片验收。

结论：**底层几何与方法论确已成熟（≈80%）；本次 P0 补的是「理解真实生活」的数据模型与
把它接进产品行为的那一层**——这层此前基本是空的。矩阵随实现推进逐条勾勒测试 ID。
