# Finance OS IA 树测试任务脚本

> 用于验证混合双层 IA 是否比旧结构更易找到目标页。每方案建议 ≥30 人；预试 ≥8 人。

## 测试准备

1. 打开 `IA_ROUTE_MAPPING.csv` 作为节点对照表。
2. 为受测者展示**纯文字导航树**（不含 UI），四方案可选：扁平行动台 / 主题领域 / 旅程导向 / **混合双层（现网）**。
3. 记录：首击节点、最终选中节点、耗时、是否直接成功。

## 关键任务（10 项）

| #   | 任务描述（读给受测者）    | 正确路径（混合双层）   | 成功判据                    |
| --- | ------------------------- | ---------------------- | --------------------------- |
| 1   | 更新某个银行账户的余额    | 资金 → 账户            | 选中 `accounts`             |
| 2   | 查看今天可以安心花多少钱  | 首页 → 今日            | 选中 `home/today`           |
| 3   | 查看净资产与长期目标全景  | 首页 → 总览            | 选中 `home/overview`        |
| 4   | 登记工资或房租等固定收支  | 资金 → 记录 → 固定收支 | 选中 `history/fixed`        |
| 5   | 登记一笔未来大额支出      | 资金 → 记录 → 大额收支 | 选中 `history/oneoff`       |
| 6   | 导入 CSV 并开始审查       | 审查 → 导入            | 选中 `review/import`        |
| 7   | 处理审查队列里的异常交易  | 审查 → 队列            | 选中 `review/queue`         |
| 8   | 看未来 6 个月现金曲线低谷 | 规划 → 预测            | 选中 `forecast/forecast`    |
| 9   | 对比两个消费情景并保存    | 规划 → 决策 → 对比     | 选中 `decision/compare`     |
| 10  | 修改通胀或工资增长假设    | 设置 → 预测参数        | 选中 `settings/assumptions` |

## KPI 目标（来自 IA 调研报告）

| 指标                         | 目标              |
| ---------------------------- | ----------------- |
| 树测试成功率（上述 10 任务） | ≥ 80%             |
| 平均查找时间                 | 比基线下降 15–25% |
| 首击正确率                   | +10–20%           |
| 误入后回退率                 | −20%              |

## 受测后分析

- 失败任务按「命名 / 层级 / 域边界」三类归因。
- 搜索日志（若有）中「更新账户」「导入」「决策对比」高频词应对照导航缺口。
- 结果写回 `INFORMATION_ARCHITECTURE.md` 变更记录。
