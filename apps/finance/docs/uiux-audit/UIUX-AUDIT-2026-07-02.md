# Finance OS UI/UX 审查与优化建议

> 审查日期：2026-07-02
> 方法：使用 Playwright 在本地 dev（`localhost:5199`）以测试账号登录，录入演示数据（4 个账户、3 项固定收支、2 笔大额收支，另有 7 笔历史交易与内置持仓快照），对全部 8 个主页面 + 子页 + 抽屉分别在桌面端（1440×900）与移动端（390×844）截图走查。
> 截图：见 `docs/uiux-audit/screenshots/`（`desktop-*` 19 张，`mobile-*` 10 张）。

---

## 一、现状总结：这个产品的 UIUX 做对了什么

先说优点，这些是后续优化时必须保留的资产：

1. **信息架构（IA）有清晰的心智模型**：侧边栏按「日常 → 资产 → 收支 → 前瞻」分组，每页开头都有一句"这一页只看什么、别的去哪看"的定位说明，这在个人财务工具里非常少见，能显著降低迷路感。
2. **空状态引导好**：首次进入「今日」的欢迎卡（`desktop-today-empty.png`）直接给出两个 CTA（录入账户 / 录入每月收支），指向明确。
3. **上下文 FAB 设计聪明**：FAB 随页面切换（今日=试算消费、洞察=记一笔、大额收支=快速录入），减少全局菜单负担。
4. **口径透明**：几乎每个数字都有 tooltip 解释计算逻辑（如「未来低谷余量可花」的完整推导），体现了"可信"这一核心价值。
5. **移动端已有基础适配**：底部 tabbar、洞察页把图表折叠为「展开深入分析」、`safe-area` 变量都已就位。

## 二、全局性问题（跨页面，优先修）

### P0-1 收入符号约定错误/混乱 —— 数据可信度硬伤

多处把收入显示为**负数甚至红色**：

- 记录→洞察 KPI：「近 12 月平均月收入 **-$2,800**」（`desktop-records-insights.png`）
- 流水列表：`PAYROLL ACME / Income` 显示为红色 **-$2,800**
- 审查→消费基线：「月收入 -$2,800」「月净现金流 -$4,129」（实际是净结余为正）

对一个以"可信基线"为卖点的财务工具，符号错乱是第一优先级 bug。约定应统一为：**收入=正/绿，支出=负/红**，并全链路（导入、展示、聚合）核对一遍。

### P0-2 核心数字互相矛盾，用户无法建立信任

「今日」页同屏出现（`desktop-today.png`）：

- 未来低谷余量可花 **$0**（并配警告"暂无可自由支配现金"）
- 本月预计能存 **$12,274**

两个大字号数字一个说"没钱花"、一个说"能存一万二"，普通用户无法调和。根因是两个口径（低谷余量 vs 存款计划执行额），且存款滑块默认拉满到 $12,274（超出每月结余 $2,145，需动用存量现金），导致「可花余量」被目标预留吃光。建议：

- 存款滑块默认值改为「每月结余」而非最大可行值；
- 当「可花 $0」是因为用户自己设置的存款目标导致时，明确说"已被你的存款计划占用 $X"，并给「调整存款计划」的直接入口，而不是笼统的"考虑放慢大额支出"警告。

### P0-3 内部术语/英文行话直接泄漏到界面

| 出现位置 | 原文 |
|---|---|
| 总览 | 「Safe-to-spend 拆解」「earmark 目标现金」「protected reserve（已上游排除）」 |
| 试算抽屉 | 「safe-after 怎么算?」 |
| 审查队列表格 | `seed review` / `high` / `confirm exclude` / `open`（整列原始枚举值） |
| 资产配置 | 「Partial」（分析完整度徽章） |
| 设置 | 「导出为 schema v2」 |
| 预测/长期规划 | 「已 earmark 目标」 |

这些是工程词汇，应全部翻译成用户语言（例：Safe-to-spend → 「现在能放心花」——产品里其实已有这个词，说明是同一概念两套叫法，需要统一术语表）。

### P0-4 无 URL 路由，刷新即丢状态

所有导航都是 `useState`（`AppShell.tsx` 的 `tab` state），刷新永远回到「今日」，无法深链、无法用浏览器前进/后退。移动端影响更大：手机浏览器/PWA 的系统级后退手势会直接退出应用而不是返回上一页。建议引入 hash 路由（`#/history/insights`）即可，改造成本低、不需要 SSR。

### P1-5 「key-value 行」过度使用，缺少可视化层级

总览页（`desktop-overview.png`）几乎全部内容是两栏文本行——「钱都在哪里」区块连续 11 行 label:value，包含成本基础、估税、税后等中间计算过程。资产配置页同样非常长。建议：

- 每个卡片只保留 1-3 个**结论数字**，中间推导过程收进「查看计算明细」折叠区；
- 净资产构成、能动/不能动比例用一条堆叠条形图或环形图表达，比 11 行文字快 10 倍；
- 桌面端 1440px 宽度下大量卡片仍是通栏单列，右侧留白浪费，可用 2 列网格。

### P1-6 重复入口过多

「试算消费」在今日页有 3 个入口（FAB、"算一笔消费的影响"按钮、更多工具区），总览页又有 1 个；「设定目标」在资产配置页出现 4 次（快捷栏、目标与偏离、P1 建议、预填按钮）。重复入口让页面显得杂乱且稀释视觉优先级——每页每个动作保留 1 个主入口即可。

### P1-7 决策工作室：步骤跳跃 + 货币单位错误

- 界面直接显示「第 1 步 → 第 4 步 → 第 5 步」（`desktop-decision.png`），第 2/3 步在预览前不存在，用户会以为界面坏了。应改为渐进式披露：先只显示第 1 步，预览后再展开后续，或改用完整的 wizard 分步组件。
- 金额单位显示「**元**」，与全站的 `$` 不一致。

### P1-8 审查→消费基线：三张完全相同的卡片

3 个月/6 个月/12 个月基线在数据不足时显示完全一样的数字（`desktop-review-baseline.png`），且标着「尚不可用」却仍显示大字号数字。数据不足时应显示一张卡 + "还需 N 个完整月才能区分 3/6/12 月基线"，避免复制三份噪音。

### P2-9 其他小问题

- 洞察页「月度花销 vs 收入」只有 1 个月数据时，柱状图渲染成一根占满全宽的巨柱（`desktop-records-insights.png`），应设置最大柱宽。
- Console 有多个 recharts `width(-1)/height(-1)` 警告：隐藏 tab 中的图表仍在渲染，既是性能浪费也是警告来源（StocksView 用 `hidden` 挂载所有 tab）。
- 信用卡编辑表单出现**两个重复的「年利率 APR」字段**（设置→账户→信用卡展开态）。
- 全部流水列表金额列：信用卡还款显示 `—` 但仍有编辑/删除按钮占位，行右侧永远挤着两个按钮，视觉噪音大。
- 登录页风格与主应用完全不同（深色玻璃拟态 vs 主应用米白纸质感），品牌不连贯。

## 三、移动端专项审查

### 3.1 当前移动端的主要问题

**A. 底部 Tabbar 塞了 8 个入口（最严重）**
`mobile-today-viewport.png` 可见：今日/总览/资产配置/记录/审查/预测/决策/设置全部挤在 390px 宽度里，每项约 48px，「资产配置」标签换行成两行，触摸目标拥挤（iOS HIG 建议 tab 不超过 5 个）。CSS 里 tabbar 允许横向滚动但滚动条被隐藏，用户不知道能滑。

**建议**：移动端 tabbar 收敛为 4+1：

| Tab | 内容 |
|---|---|
| 今日 | 保持 |
| 资产 | 总览（内含资产配置入口） |
| 记录 | 洞察/固定/大额 |
| 预测 | 预测曲线/长期规划 |
| 更多 | 审查、决策、设置（低频页面） |

审查（CSV 导入）和决策（多步 wizard）本质是桌面流程，移动端放「更多」里完全够用。

**B. Hover tooltip 在触屏上不可用**
全站大量 `?` 图标依赖 hover 展示口径解释（今日页就有 5 个），触屏无 hover，等于移动端用户完全看不到这些关键解释。改为点按弹出 popover/bottom sheet，或把最重要的一句话直接写在卡片副标题里。

**C. 超长单列页面无页内导航**
移动端总览约 9 屏长、资产配置约 12 屏长（`mobile-overview-full.png`、`mobile-stocks-full.png`），全部靠滚动。建议：

- 移动端把次要区块默认折叠（洞察页的「展开深入分析」模式已经证明可行，推广到总览/资产配置）；
- 长页顶部加锚点 chips（如资产配置页：快照/诊断/目标/持仓）。

**D. 横向溢出与截断**
预测页口径切换 chips（能动的钱/流动资产/净资产/投资资产/不能动的钱）在 390px 下溢出且无渐变提示（`mobile-forecast.png` 中「不能动的钱」被切掉一半）；预测 KPI 长数字串折行别扭。可横滚的行需要加渐隐边缘或换行成两行 chips。

**E. 列表行操作按钮常驻**
移动端流水列表每行都常驻「编辑/删除」两个按钮（`mobile-records-insights.png`），既挤压金额列又容易误触删除。改为：整行可点进详情/编辑（bottom sheet），删除收进详情或用左滑操作，并加确认。

**F. FAB 为带文字的胶囊按钮，遮挡内容**
FAB 悬浮在 tabbar 上方 14px，滚动到列表底部时会盖住最后一行内容。移动端可改为纯图标圆形 FAB，或滚动时收缩为图标。

### 3.2 哪些 mobile flow 需要特别设计与强化

手机是"发生现场"的设备（正在店里、刚收到账单、刚看完银行 App），移动端应该围绕**高频、短时、单手**的场景重新设计以下 flow，而不是把桌面页面缩窄：

**Flow 1：快速记一笔（最高优先）**
现状：「记一笔」抽屉有 6 个字段，商户/类别/账户全是自由文本输入（`desktop-txn-drawer.png`），手机上打字成本极高。
目标：10 秒内完成。设计要点：

- 金额放第一位、自动聚焦、调起数字键盘（`inputMode="decimal"`）；
- 类别改为最近使用 + 常用类别的 chips 网格（一次点击）；
- 商户提供历史商户自动补全；账户默认上次使用；
- 日期默认今天，只在需要时展开；
- 保存后 toast + 停留在原地可连续记账。

**Flow 2：今日现金一瞥（打开即答案）**
现状：打开 app 首屏要读一段定位文字 + 两张互相矛盾的卡才能知道"我现在能花多少"。
目标：首屏（above the fold）一眼看到——①现在能放心花的钱 ②下一笔账单（日期+金额）③下次发薪。其余（对账提示、存款计划、更多工具）全部下移或折叠。这是移动端最核心的价值主张："排队买咖啡时掏出手机看一眼"。

**Flow 3：账单提醒 → 付款闭环**
现状已有「本月关键账单 + 付款链接」（跳转 Amex 登录页），这是好雏形。强化：

- 账单临近时的可视倒计时（"3 天后 -$980"）；
- 付款后一键标记「已还」，与现金日历联动；
- 配合 PWA 推送（见 Flow 7）做还款日提醒。

**Flow 4：试算消费（分享/冲动场景）**
「看到想买的东西马上算一下」天然发生在手机上。现状抽屉已可用（`mobile-spend-drawer.png`），强化：

- 结论前置：把「影响不大，可以买」的绿色结论从底部提到金额输入框正下方，输入即见；
- 快捷金额 chips（$100/$500/$1000/自定义）；
- 修复「safe-after 怎么算?」术语。

**Flow 5：手机对账（抄银行余额）**
对账的输入源是银行 App——用户几乎一定是在手机上左右切换两个 App 完成的。现状对账表单藏在「审查」第 5 个 tab 里。强化：

- 在「今日」的对账提示卡直接内嵌"输入银行余额"输入框，一步完成；
- 每个账户行加「快速对账」入口；
- 未来可用 iOS/Android 剪贴板检测（复制了金额自动填入）。

**Flow 6：CSV 导入/审查队列 —— 明确降级为桌面流程**
手机上选 CSV 文件、逐行审查交易的体验注定差。移动端「审查」页应显示："导入和批量审查建议在电脑上完成"，只保留**审查队列的确认/忽略**这种单手可做的轻操作（做成卡片式一次一条的 swipe 流，类似邮件分类），把 6 步导入向导藏起来。

**Flow 7：PWA 化（承载以上所有 flow 的基础设施）**
项目已按 1 电脑 + 1 手机的设备模型设计（设备槽管理），但目前没有 manifest/service worker。建议：

- 加 Web App Manifest（图标、`display: standalone`、主题色）支持添加到主屏幕；
- Service worker 缓存 shell + 最近数据（AuthGate 已有本地缓存秒开逻辑，离手离线可用只差一步）；
- 后续可加推送：账单提醒、发薪日存钱提醒（对应今日页「最佳存钱日」）。

### 3.3 移动端表单与交互规范建议

- 所有编辑（账户、固定收支、大额收支）从"列表内联展开"改为 **bottom sheet**：内联展开在 390px 下把列表撑得很长且丢失上下文；
- 数字输入统一 `inputMode="decimal"`，百分比/金额输入框右侧固定单位；
- 触摸目标 ≥44×44px（现 tabbar 项、流水行按钮、`?` 图标均不达标）；
- 长横滚区域（预测 chips、审查子 tab、参考框架按钮组）统一加渐隐边缘提示。

## 2026-07-02 落地记录（v2）

同日已实施以下优化（截图见 `screenshots/v2-*.png`）：

### 移动端预算 / 记账
- **快速记一笔重做**（`TxnEntryDrawer`）：金额优先大输入（数字键盘）、花销/收入/退款切换、类别 chips（最近常用类别自动排前）、今天/昨天日期 chips、商户/账户折叠为可选项（带最近建议）、「保存并继续」连续记账。
- **预算脉搏卡**（`BudgetPulseCard`，记录→洞察顶部）：本月已花 vs 预算进度条（含时间进度参考线与超速/低于进度着色）、今日已花、剩余日均可花、近 7 日每日花销柱。预算基准 = 固定收支里的月度支出计划。
- **FAB 全局化**：今日/总览/记录页 FAB 均为「记一笔」，随处可快速记账。
- **移动 tabbar 收敛**：8 项 → 今日/总览/记录/预测 + 「更多」bottom sheet（资产配置/审查/决策/设置）。
